# 📦 庫存管理系統整合指南

## 📋 整合概述

將原本分離的「物品管理」和「庫存查詢」兩個模組整合為統一的「物品與庫存管理」介面，提供更友善的使用者體驗和更高效的操作流程。

### 整合日期
2025-10-04

### 整合原因
1. **減少重複性**：兩個模組功能重疊，造成使用者困惑
2. **提升效率**：使用者不需在兩個頁面間切換
3. **資料整合**：在同一介面查看物品資訊和庫存狀態
4. **操作便利**：直接在列表頁進行入庫、出庫等操作

## 🔧 故障排除

### 問題 1：選單沒有顯示「物品與庫存管理」

**解決方案**：
1. 確認資料庫 SQL 已執行成功
2. 清除瀏覽器快取
3. 重新登入系統
4. 檢查角色權限是否正確設定

### 問題 2：API 請求失敗 (404)

**解決方案**：
1. 確認後端已重新編譯
2. 確認 `InvManagementController.java` 已正確編譯
3. 檢查後端日誌是否有錯誤訊息
4. 確認 Spring Boot 應用已重新啟動

### 問題 3：前端頁面顯示異常

**解決方案**：
1. 確認前端已重新編譯：`npm run build:prod`
2. 檢查瀏覽器 Console 是否有 JavaScript 錯誤
3. 確認路由配置正確
4. 清除瀏覽器快取並重新載入

### 問題 4：庫存狀態計算不正確

**解決方案**：
1. 檢查 `inv_item` 表的 `min_stock` 欄位是否有正確設定
2. 檢查 `inv_stock` 表的資料是否正常
3. 確認 MyBatis XML 的 SQL 查詢正確
4. 檢查 DTO 的 `calculateStockStatus()` 方法邏輯

### Step 1: 復原資料庫選單

```sql
-- 顯示原有選單
UPDATE sys_menu 
SET visible = '0', order_num = 1, 
    remark = REPLACE(remark, ' [已整合至物品與庫存管理]', '')
WHERE menu_name IN ('物品管理', '庫存查詢');

-- 刪除整合選單
DELETE FROM sys_menu WHERE menu_name = '物品與庫存管理';
DELETE FROM sys_menu WHERE perms LIKE 'inventory:management:%';
```

