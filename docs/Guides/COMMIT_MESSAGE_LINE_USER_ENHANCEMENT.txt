## 功能說明

### 1. 關注狀態完整支援
- **UNFOLLOWED**: 新使用者預設狀態或曾經取消關注
- **FOLLOWING**: 使用者加入好友（follow 事件）
- **BLOCKED**: 使用者取消關注或封鎖（unfollow 事件）

### 2. 自動解除綁定機制
當使用者取消關注時（unfollow 事件）：
- 自動將 bindStatus 改為 UNBOUND
- 清空 sysUserId（解除與系統使用者的關聯）
- 記錄 unbindTime 和 blockTime
- 確保使用者資料一致性

### 3. 匯入功能增強
- 下拉選單只顯示啟用的 LINE 頻道
- 欄位對應正確（configId）
- 使用專用 API 提升效能

## 狀態轉換流程

```
新建立使用者 → UNFOLLOWED (未關注)
       ↓
  follow 事件
       ↓
    FOLLOWING (關注中)
       ↓
  unfollow 事件
       ↓
    BLOCKED (已封鎖) + 自動解除綁定
```
