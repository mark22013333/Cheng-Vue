# 圖片顯示問題修正完成總結

## 📋 任務概述

**問題**：Zeabur 部署後，前端圖片顯示為 `src="undefined/profile/avatar/..."` 

**影響範圍**：
- 使用者頭像
- 庫存管理物品圖片
- 掃描頁面物品圖片

## ✅ 已完成的修正

### 1. 核心問題修正

#### 問題根源
1. `process.env.VUE_APP_BASE_API` 在某些情況下為 `undefined`
2. 後端返回的路徑已經是 `/profile/xxx` 格式，不需要再加前綴
3. `.dockerignore` 排除了 `.env.production`，導致建置時環境變數未載入

#### 解決方案

**a. 建立統一圖片處理工具** ✅
- 檔案：`cheng-ui/src/utils/image.js`
- 功能：統一處理所有圖片路徑格式
- 防禦性處理 `undefined`：使用 `|| ''` 預設值

**b. 修正使用者頭像邏輯** ✅
- 檔案：`cheng-ui/src/store/modules/user.js`
- 修正：判斷路徑是否以 `/profile` 開頭
- 結果：避免重複加前綴

**c. 修正圖片上傳組件** ✅
- 檔案：`cheng-ui/src/components/ImageUpload/index.vue`
- 修正：`process.env.VUE_APP_BASE_API || ''`
- 結果：防止 `undefined` 出現在路徑中

**d. 修正業務頁面** ✅
- `views/inventory/management/index.vue` - 使用統一工具
- `views/inventory/scan/index.vue` - 使用統一工具

**e. 修正 Docker 建置配置** ✅
- 檔案：`cheng-ui/.dockerignore`
- 修正前：`.env*` （排除所有 .env 檔案）
- 修正後：只排除本地開發檔案，保留 `.env.production`

## 📊 技術細節

### 圖片路徑處理邏輯

```javascript
function getImageUrl(imagePath) {
  // 1. 空值檢查
  if (!imagePath) return '';
  
  // 2. HTTP/HTTPS 絕對 URL
  if (imagePath.startsWith('http')) return imagePath;
  
  // 3. 已經是 /profile 開頭的路徑
  if (imagePath.startsWith('/profile')) {
    const baseApi = process.env.VUE_APP_BASE_API || '';
    return baseApi && !imagePath.startsWith(baseApi) 
      ? baseApi + imagePath 
      : imagePath;
  }
  
  // 4. 其他情況加上前綴
  return (process.env.VUE_APP_BASE_API || '') + '/profile/' + imagePath;
}
```

### Nginx 代理配置

```nginx
# 靜態資源代理（必須在 /prod-api/ 之前）
location /profile/ {
  proxy_pass http://coolapps-api.zeabur.internal:8080/profile/;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  
  # 快取設定（圖片可以快取較長時間）
  expires 30d;
  add_header Cache-Control "public, immutable";
}
```

### 部署後必須做的事
1. **清除瀏覽器快取**：Ctrl+Shift+Delete

### 可能遇到的問題

**問題 1：圖片仍顯示 404**
- 檢查後端 `/profile/` 路徑配置
- 確認檔案確實存在於後端伺服器

**問題 2：路徑仍有 `undefined`**
- 清除瀏覽器快取
- 確認 Zeabur 已部署最新版本
- 檢查 `.env.production` 是否包含在建置中

**問題 3：上傳後無法顯示**
- 檢查後端儲存路徑權限
- 確認 Nginx 代理配置正確
