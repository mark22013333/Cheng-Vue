# 任務類型動態配置整合說明

## 📋 架構概述

### 設計理念

- **模組解耦**：各模組（爬蟲、推播、報表）獨立定義自己的任務類型
- **動態註冊**：Spring 自動掃描並註冊所有 `TaskTypeProvider` 實作
- **參數可配置**：每個任務類型可定義支援的參數及其元數據
- **前端友善**：提供統一 API，前端根據元數據動態產生表單

---

## 🏗️ 模組結構

```
cheng-common (共用模組)
├── enums/
│   ├── TaskCategory.java           # 任務分類（爬蟲、推播、報表等）
│   └── TaskParamType.java          # 參數類型（字串、數字、下拉選單等）
├── core/domain/
│   ├── TaskParamMetadata.java      # 參數元數據
│   └── TaskTypeOption.java         # 任務類型選項
└── core/provider/
    └── TaskTypeProvider.java       # 任務類型提供者介面

cheng-crawler (爬蟲模組)
└── config/
    └── CrawlerTypeProvider.java    # 實作：提供爬蟲任務類型

cheng-quartz (排程模組)
├── controller/
│   └── SysJobController.java       # API：取得任務類型選項
└── provider/
    └── NotificationTypeProvider.java  # 實作：提供推播任務類型
```

## 🎨 前端整合

### 1. API 定義

```javascript
// src/api/monitor/job.js

import request from '@/utils/request'

/**
 * 取得所有任務類型選項
 */
export function getTaskTypes() {
  return request({
    url: '/monitor/job/taskTypes',
    method: 'get'
  })
}

/**
 * 取得特定分類的任務類型
 */
export function getTaskTypesByCategory(category) {
  return request({
    url: `/monitor/job/taskTypes/${category}`,
    method: 'get'
  })
}
```

### 2. Vue 組件範例

```vue
<template>
  <el-dialog title="新增排程任務" :visible.sync="open">
    <el-form :model="form" ref="form" :rules="rules">
      <!-- 任務分類選擇 -->
      <el-form-item label="任務分類" prop="category">
        <el-select v-model="form.category" @change="onCategoryChange">
          <el-option
            v-for="cat in categories"
            :key="cat.code"
            :label="cat.label"
            :value="cat.code"
          />
        </el-select>
      </el-form-item>
      
      <!-- 任務類型選擇 -->
      <el-form-item label="任務類型" prop="taskType">
        <el-select v-model="form.taskType" @change="onTaskTypeChange">
          <el-option
            v-for="task in currentTasks"
            :key="task.code"
            :label="task.label"
            :value="task.code"
          />
        </el-select>
      </el-form-item>
      
      <!-- 動態參數表單 -->
      <div v-if="currentParams && currentParams.length > 0">
        <el-divider content-position="left">任務參數</el-divider>
        
        <el-form-item
          v-for="param in sortedParams"
          :key="param.name"
          :label="param.label"
          :prop="`params.${param.name}`"
          :rules="buildParamRules(param)"
        >
          <!-- 字串輸入 -->
          <el-input
            v-if="param.type === 'STRING'"
            v-model="form.params[param.name]"
            :placeholder="param.description"
          />
          
          <!-- 數字輸入 -->
          <el-input-number
            v-else-if="param.type === 'NUMBER'"
            v-model="form.params[param.name]"
            :min="param.validation?.min"
            :max="param.validation?.max"
            :placeholder="param.description"
          />
          
          <!-- 下拉選單 -->
          <el-select
            v-else-if="param.type === 'SELECT'"
            v-model="form.params[param.name]"
            :placeholder="param.description"
          >
            <el-option
              v-for="opt in param.options"
              :key="opt.value"
              :label="opt.label"
              :value="opt.value"
            >
              <span>{{ opt.label }}</span>
              <span v-if="opt.description" style="color: #8492a6; font-size: 13px">
                - {{ opt.description }}
              </span>
            </el-option>
          </el-select>
          
          <!-- 日期選擇 -->
          <el-date-picker
            v-else-if="param.type === 'DATE'"
            v-model="form.params[param.name]"
            type="date"
            value-format="yyyy-MM-dd"
            :placeholder="param.description"
          />
          
          <!-- 文字區域 -->
          <el-input
            v-else-if="param.type === 'TEXTAREA'"
            v-model="form.params[param.name]"
            type="textarea"
            :rows="3"
            :placeholder="param.description"
          />
        </el-form-item>
      </div>
    </el-form>
  </el-dialog>
</template>

<script>
import { getTaskTypes } from '@/api/monitor/job'

export default {
  data() {
    return {
      open: false,
      categories: [],
      allTasks: {},
      form: {
        category: '',
        taskType: '',
        params: {}
      },
      currentParams: [],
      selectedTask: null
    }
  },
  
  computed: {
    // 當前分類的任務列表
    currentTasks() {
      return this.allTasks[this.form.category] || []
    },
    
    // 排序後的參數列表
    sortedParams() {
      return [...this.currentParams].sort((a, b) => a.order - b.order)
    }
  },
  
  methods: {
    // 載入任務類型資料
    async loadTaskTypes() {
      const { data } = await getTaskTypes()
      this.categories = data.categories
      this.allTasks = data.tasks
    },
    
    // 分類變更
    onCategoryChange() {
      this.form.taskType = ''
      this.form.params = {}
      this.currentParams = []
      this.selectedTask = null
    },
    
    // 任務類型變更
    onTaskTypeChange() {
      const task = this.currentTasks.find(t => t.code === this.form.taskType)
      if (task) {
        this.selectedTask = task
        this.currentParams = task.params || []
        
        // 設定預設值
        this.form.params = {}
        this.currentParams.forEach(param => {
          if (param.defaultValue) {
            this.form.params[param.name] = param.defaultValue
          }
        })
      }
    },
    
    // 建立參數驗證規則
    buildParamRules(param) {
      const rules = []
      
      if (param.required) {
        rules.push({
          required: true,
          message: `請輸入${param.label}`,
          trigger: 'blur'
        })
      }
      
      if (param.validation) {
        const v = param.validation
        
        // 正則驗證
        if (v.pattern) {
          rules.push({
            pattern: new RegExp(v.pattern),
            message: v.message || `${param.label}格式不正確`,
            trigger: 'blur'
          })
        }
        
        // 長度驗證
        if (v.minLength || v.maxLength) {
          rules.push({
            min: v.minLength,
            max: v.maxLength,
            message: v.message || `長度必須在 ${v.minLength}-${v.maxLength} 之間`,
            trigger: 'blur'
          })
        }
      }
      
      return rules
    },
    
    // 提交表單
    submitForm() {
      this.$refs.form.validate(valid => {
        if (valid) {
          const jobData = {
            jobName: this.selectedTask.label,
            invokeTarget: this.buildInvokeTarget(),
            // ... 其他欄位
          }
          
          // 呼叫新增 API
          console.log('提交任務:', jobData)
        }
      })
    },
    
    // 建立呼叫目標
    buildInvokeTarget() {
      const task = this.selectedTask
      const params = Object.entries(this.form.params)
        .map(([key, value]) => `'${value}'`)
        .join(', ')
      
      return `${task.beanName}.${task.defaultMethod}('${this.form.taskType}', ${params})`
    }
  },
  
  created() {
    this.loadTaskTypes()
  }
}
</script>
```

## 📝 總結

### 優點

1. **模組獨立**：各模組定義自己的任務類型，不互相依賴
2. **動態配置**：參數可以動態定義，前端自動產生表單
3. **類型安全**：使用 Enum 和元數據，避免字串拼錯
4. **易於擴展**：新增任務類型只需實作 `TaskTypeProvider`
