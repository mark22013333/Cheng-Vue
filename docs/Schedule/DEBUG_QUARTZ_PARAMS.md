# 定時任務參數驗證 Debug 指南

## 📋 問題說明

如果您在提交任務時看到「必填參數」的錯誤提示，請按照以下步驟進行診斷。

---

## 🔍 重要澄清

### 欄位標記 vs 驗證錯誤

**欄位標記（正常）**：
```
爬蟲類型: [CA102_________] STRING
⭐ 必填                         ← 這是灰色的標記，不是錯誤
```

**驗證錯誤（問題）**：
```
彈出紅色訊息框：
「爬蟲類型 為必填參數」          ← 這才是真正的錯誤
```

---

## 🛠️ Debug 步驟

### 步驟 1：打開瀏覽器開發者工具

1. 在瀏覽器中按 `F12` 或右鍵選擇「檢查」
2. 切換到「Console」標籤
3. 清空之前的日誌（垃圾桶圖標）

### 步驟 2：重新操作

1. 點選「新增任務」
2. 選擇任務類型（如：執行爬蟲（完整參數））
3. 觀察 Console 中的日誌

**預期看到的日誌**：
```
開始初始化參數預設值...
參數 crawlerType - 範例值: CA102
已設定 crawlerType = CA102
參數 mode - 範例值: full-sync
已設定 mode = full-sync
參數 batchSize - 範例值: 1000
已設定 batchSize = 1000
參數 timeout - 範例值: 120000
已設定 timeout = 120000
參數初始化完成，taskParams: {crawlerType: "CA102", mode: "full-sync", batchSize: "1000", timeout: "120000"}
```

### 步驟 3：嘗試提交

1. 點選「確定」按鈕
2. 觀察 Console 中的驗證日誌

**預期看到的日誌**：
```
開始驗證參數...
當前參數: {crawlerType: "CA102", mode: "full-sync", batchSize: "1000", timeout: "120000"}
需要驗證的參數列表: [...]
驗證參數 crawlerType: CA102 required: true
參數 crawlerType 驗證通過
驗證參數 mode: full-sync required: true
參數 mode 驗證通過
驗證參數 batchSize: 1000 required: false
驗證參數 timeout: 120000 required: false
所有參數驗證通過
```

### 步驟 4：分析問題

#### 情況 A：參數初始化失敗

**日誌顯示**：
```
參數 crawlerType - 範例值: undefined    ← 問題：沒有範例值
```

**解決方法**：
檢查後端 API 是否正確回傳 `example` 值

#### 情況 B：參數未綁定

**日誌顯示**：
```
當前參數: {}                             ← 問題：參數物件是空的
```

**可能原因**：
1. taskParams 沒有正確初始化
2. Vue 響應式問題

**解決方法**：
檢查是否使用了 `this.$set()` 來設定參數

#### 情況 C：驗證邏輯錯誤

**日誌顯示**：
```
驗證參數 crawlerType: CA102 required: true
參數 crawlerType 驗證失敗！值為: CA102   ← 問題：有值卻驗證失敗
```

**可能原因**：
驗證邏輯有問題

- 灰色顯示
- 改為簡潔的標記

### 修復 2：驗證邏輯

**改前**：
```javascript
if (param.required && !this.taskParams[param.name]) {
  // 會把 0、false 都當作空值
}
```

**改後**：
```javascript
const value = this.taskParams[param.name]
if (param.required && (value === undefined || value === null || value === '')) {
  // 只有真正為空時才報錯
}
```

新增詳細的 console.log，方便診斷問題：
- 參數初始化過程
- 參數驗證過程
- 每個參數的值

## 🧪 測試方法

### 測試 1：基本提交

1. 選擇任務類型：「執行爬蟲（無參數）」
2. 檢查參數：應該自動填入 CA102
3. 點選「確定」
4. **預期結果**：成功提交，無錯誤

### 測試 2：修改參數後提交

1. 選擇任務類型：「執行爬蟲（無參數）」
2. 修改參數：將 CA102 改為 CA103
3. 點選「確定」
4. **預期結果**：成功提交，呼叫目標為 `crawlerTask.run('CA103')`

### 測試 3：清空必填參數

1. 選擇任務類型：「執行爬蟲（無參數）」
2. 清空參數：將 CA102 刪除
3. 點選「確定」
4. **預期結果**：彈出錯誤訊息「爬蟲類型 為必填參數」

### 測試 4：數字參數

1. 選擇任務類型：「執行爬蟲（完整參數）」
2. 將批次大小設為 0
3. 點選「確定」
4. **預期結果**：成功提交（0 是有效值）

## 🎯 常見問題

### Q1: 為什麼參數下方顯示「必填」？

**A**: 這是欄位標記，不是錯誤。它告訴您這個欄位是必填的。

### Q2: 什麼時候會出現真正的錯誤？

**A**: 當您點選「確定」後，如果必填參數為空，才會彈出紅色錯誤訊息框。

### Q3: 如何知道參數是否正確填入？

**A**: 
1. 查看表單中的輸入框是否有值
2. 查看「呼叫目標」欄位是否正確產生
3. 查看 Console 中的 debug 日誌

### Q4: 參數值改為 0 會報錯嗎？

**A**: 不會。數字 0 是有效值，驗證邏輯已修復。
