# 程式碼審查清單

> **目的**: 開發完成後自我檢查，確保程式碼品質

---

## ⭐ Enum 使用檢查

- [ ] 是否有使用魔法數字或字串？
- [ ] 所有狀態/類型是否都使用 Enum？
- [ ] Enum 是否包含 `code` 和 `description`？
- [ ] Enum 是否加上 `@JsonValue` 註解？
- [ ] Enum 是否提供 `getByCode()` 方法？
- [ ] 是否建立對應的 Spring Converter？
- [ ] 資料庫欄位是否儲存 code 而非 name？

**範例檢查**:
```java
// ❌ 不好
if (status.equals("0")) { ... }

// ✅ 好
if (status == Status.NORMAL) { ... }
```

---

## 🔧 效能優化檢查

### 資料庫查詢

- [ ] 所有列表查詢都使用分頁？
- [ ] 是否避免 N+1 查詢問題？
- [ ] 是否有適當的資料庫索引？
- [ ] 慢查詢數是否為 0？（< 1 秒）

### 快取使用

- [ ] 熱點資料是否使用 Redis 快取？
- [ ] 快取是否設定合理的過期時間？
- [ ] 更新資料時是否刪除快取？
- [ ] 是否防止快取穿透？

### 並發控制

- [ ] 關鍵操作是否有並發控制？
- [ ] 是否使用 `@RepeatSubmit` 防重複提交？
- [ ] 高頻 API 是否使用 `@RateLimiter` 限流？
- [ ] 庫存等關鍵資料是否使用樂觀鎖？

---

## ⚠️ 異常處理檢查

- [ ] 是否統一使用 `ServiceException`？
- [ ] 異常訊息是否清晰明確？
- [ ] 異常訊息是否使用繁體中文？
- [ ] 異常訊息是否包含關鍵資訊？
- [ ] 是否告知用戶如何解決？
- [ ] 異常是否被正確處理（不要吞掉）？
- [ ] 資源是否正確釋放（try-finally）？
- [ ] 是否洩漏敏感資訊？

**範例檢查**:
```java
// ❌ 不好
throw new ServiceException("刪除失敗");

// ✅ 好
throw new ServiceException(
    String.format("分類「%s」正被 %d 個物品使用，無法刪除", 
        categoryName, itemCount)
);
```

---

## 🔄 事務管理檢查

- [ ] 多表操作是否加上 `@Transactional`？
- [ ] 是否加上 `rollbackFor = Exception.class`？
- [ ] 是否避免大事務（< 3 秒）？
- [ ] 非核心邏輯是否移到事務外？
- [ ] `@Transactional` 是否加在 public 方法？
- [ ] 是否避免同類別內部呼叫？
- [ ] 捕獲異常是否重新拋出或手動回滾？
- [ ] 唯讀操作是否加上 `readOnly = true`？

---

## 📝 日誌記錄檢查

- [ ] 重要業務操作是否有日誌記錄？
- [ ] 日誌級別是否正確？
- [ ] 是否記錄敏感資訊（密碼、Token）？
- [ ] 異常日誌是否包含堆疊追蹤？
- [ ] 是否使用參數化日誌？
- [ ] 正式環境是否關閉 DEBUG 級別？

**範例檢查**:
```java
// ❌ 不好
log.info("用戶登入, password=" + password);

// ✅ 好
log.info("用戶登入成功, userName={}", userName);
```

---

## 🌐 API 設計檢查

- [ ] 是否遵循 RESTful 規範？
- [ ] 是否使用統一的回應格式 `AjaxResult`？
- [ ] 分頁參數是否使用 `pageNum`/`pageSize`？
- [ ] 時間格式是否統一 `yyyy-MM-dd HH:mm:ss`？
- [ ] Enum 是否使用 code 序列化？
- [ ] URL 路徑是否清晰、語義化？
- [ ] 是否有 Swagger 文件註解？

---

## ✅ 資料驗證檢查

### Controller 層

- [ ] 是否加上 `@Valid` 或 `@Validated`？
- [ ] Entity 欄位是否有驗證註解？
- [ ] 驗證訊息是否使用繁體中文？

### Service 層

- [ ] 是否檢查必填欄位？
- [ ] 是否檢查唯一性？
- [ ] 是否檢查業務規則？
- [ ] 是否檢查關聯資料？
- [ ] 是否使用 `StringUtils` 判空？

---

## 🔒 安全檢查

- [ ] 敏感資訊是否使用 Jasypt 加密？
- [ ] 密碼是否使用 BCrypt 加密？
- [ ] 是否有 SQL 注入風險？（使用參數化查詢）
- [ ] 是否有 XSS 風險？（輸入輸出過濾）
- [ ] 關鍵操作是否有權限控制 `@PreAuthorize`？
- [ ] 是否有操作日誌記錄 `@Log`？
- [ ] JWT Token 是否正確處理？
- [ ] CORS 是否正確配置？

---

## 📱 前端程式碼檢查

### 基本檢查
- [ ] API 路徑是否使用環境變數 `VUE_APP_BASE_API`？
- [ ] 是否使用繁體中文（台灣用語）？
- [ ] Element UI 是否使用繁體中文語言包？
- [ ] 錯誤訊息是否清晰明確？
- [ ] 是否有 loading 狀態提示？
- [ ] 是否有適當的錯誤處理？
- [ ] 表單驗證是否完整？
- [ ] 路由是否使用懶載入？

### ⭐ 狀態參數處理（最重要）
- [ ] 後端返回的枚舉物件是否正確提取 `code`？
- [ ] 後端返回的數字是否轉為字串（el-radio-group）？
- [ ] 後端返回的 YES/NO 是否轉為布林值（el-switch）？
- [ ] 提交時字串狀態是否轉為數字？
- [ ] 提交時是否使用 `xxxCode` 欄位名？
- [ ] 是否移除原始欄位避免衝突？
- [ ] el-radio-group 的 label 是否為字串？
- [ ] el-switch 的 active/inactive-value 是否正確？
- [ ] 是否加上 console.log 確認轉換？

---

## 🗃️ 資料庫檢查

- [ ] 表名和欄位名是否使用底線分隔？
- [ ] 是否包含 `create_time`、`update_time` 等審計欄位？
- [ ] 是否包含 `del_flag` 邏輯刪除欄位？
- [ ] 主鍵是否使用 `AUTO_INCREMENT`？
- [ ] 外鍵欄位是否建立索引？
- [ ] 是否有適當的預設值？
- [ ] 註解是否完整？

---

## 🎨 程式碼風格檢查

- [ ] 類別名稱是否使用 PascalCase？
- [ ] 方法名稱是否使用 camelCase？
- [ ] 常數是否使用 UPPER_SNAKE_CASE？
- [ ] 是否有適當的 JavaDoc 註解？
- [ ] 程式碼是否有適當的空行分隔？
- [ ] import 是否整理（移除未使用）？
- [ ] 是否有無用的程式碼（註解掉的程式碼）？

- [ ] API 是否在 Swagger UI 測試過？
- [ ] 是否測試正常情況和異常情況？
- [ ] 是否測試邊界條件？
- [ ] 並發場景是否測試？

- [ ] 部署腳本是否更新？
- [ ] 環境配置說明是否更新？

- [ ] 敏感資訊是否已加密？

## ✨ 最終檢查清單

開發完成後，逐項檢查以下內容：

### 必須檢查（⭐ 最重要）
- [ ] ⭐ 是否使用 Enum 替代魔法數字？
- [ ] ⭐ 所有列表查詢都使用分頁？
- [ ] ⭐ 異常訊息是否清晰明確？
- [ ] ⭐ 多表操作是否加上事務？
- [ ] ⭐ 敏感資訊是否加密？

### 建議檢查
- [ ] 是否有適當的快取？
- [ ] 是否有並發控制？
- [ ] 是否有操作日誌？
- [ ] 是否有參數驗證？
- [ ] API 回應時間是否 < 1 秒？

### 可選檢查
- [ ] 是否有單元測試？
- [ ] 程式碼風格是否統一？
- [ ] 註解是否完整？
- [ ] 是否有重複程式碼可以抽取？

## 📋 Code Review 範本

提交 PR 時使用以下範本：

```markdown
## 變更說明
簡要說明此次變更的內容和目的

## 變更類型
- [ ] 新功能 (feat)
- [ ] Bug 修復 (fix)
- [ ] 效能優化 (perf)
- [ ] 重構 (refactor)
- [ ] 文件更新 (docs)
- [ ] 其他

## 自我檢查清單
- [ ] 程式碼遵循專案規範
- [ ] 已使用 Enum 替代魔法數字
- [ ] 查詢操作使用分頁
- [ ] 多表操作加上事務
- [ ] 異常訊息清晰明確
- [ ] 敏感資訊已加密
- [ ] 已在本地測試通過
- [ ] 已更新相關文件

## 測試說明
說明如何測試此變更

## 影響範圍
說明此變更可能影響的模組或功能

## 相關 Issue
關聯的 Issue 編號（如果有）
```
