# AI 助手工作規範

> **建立時間**: 2025-11-03  
> **重要性**: ⭐⭐⭐⭐⭐ 極高  
> **適用對象**: 所有 AI 助手（Cascade / Windsurf）

---

## 🚫 禁止事項

### ❌ 不要自動產生文件

**規則**:
- **不要**在每次完成功能後自動產生總結文件
- **不要**自動建立測試文件
- **不要**自動建立使用說明文件
- **不要**自動建立 README 或其他文件

**原因**:
1. 浪費 Token 額度
2. 產生大量不必要的文件
3. 使用者可能不需要這些文件

**正確做法**:
1. ✅ **完成功能後詢問使用者**是否需要建立文件
2. ✅ **等待使用者明確同意**後再產生
3. ✅ 只產生**使用者要求的**特定文件

---

## ✅ 允許的文件

### 可以自動產生的文件（不需詢問）

1. **程式碼內註解**
   - 函數說明
   - 參數說明
   - 返回值說明
   - 使用範例

2. **Git Commit Message**
   - 簡潔的變更說明
   - 修改的檔案列表

3. **簡短的修改說明**（在回應中）
   - 修改了什麼
   - 為什麼修改
   - 如何測試

### 需要詢問使用者的文件

以下文件**必須經過使用者同意**才能產生：

1. ❓ 功能總結文件（SUMMARY_*.md）
2. ❓ 測試文件（TEST_*.md）
3. ❓ API 文件（API_*.md）
4. ❓ 使用說明文件（GUIDE_*.md）
5. ❓ 設計文件（DESIGN_*.md）
6. ❓ 部署文件（DEPLOYMENT_*.md）
7. ❓ 任何其他獨立的 Markdown 文件

## 📝 工作流程範例

### ❌ 錯誤流程

```
1. 使用者要求：「幫我優化這個功能」
2. AI 完成功能優化
3. AI 自動建立：
   - SUMMARY_OPTIMIZATION.md（總結文件）
   - TEST_OPTIMIZATION.md（測試文件）
   - GUIDE_OPTIMIZATION.md（使用說明）
4. 結果：浪費大量 Token，產生使用者不需要的文件
```

### ✅ 正確流程

```
1. 使用者要求：「幫我優化這個功能」
2. AI 完成功能優化
3. AI 簡短回應：
   「✅ 功能優化完成！
   
   修改內容：
   - 檔案 A：優化了 XXX
   - 檔案 B：修正了 YYY
   
   測試建議：
   - 步驟 1
   - 步驟 2
   
   需要我建立詳細的技術文件嗎？」
   
4. 等待使用者回應
5. 如果使用者說「要」，才產生文件
```

## 💡 回應格式建議

### 完成功能後的標準回應

```markdown
# ✅ [功能名稱] 完成

## 修改內容
- **檔案 1**: 說明
- **檔案 2**: 說明

## 測試步驟
1. 步驟 1
2. 步驟 2

## 注意事項
- 注意事項 1
- 注意事項 2

---

需要我建立以下文件嗎？
- [ ] 詳細技術文件
- [ ] 測試案例文件
- [ ] API 文件
```

這樣使用者可以選擇是否需要文件，不會浪費 Token。

## 🎯 Token 使用原則

### 優先順序

1. **最高優先**: 完成使用者要求的功能
2. **次要優先**: 簡短清楚的回應說明
3. **最低優先**: 文件產生（需經過同意）

### Token 預算分配

假設每次對話有 10,000 Token 預算：

```
功能實現: 7,000 Token (70%)
回應說明: 2,000 Token (20%)
文件產生: 1,000 Token (10%) - 需經同意
```

```
使用者要求完成功能
        ↓
   完成功能實現
        ↓
   簡短回應說明
        ↓
   詢問是否需要文件？
        ↓
      ┌─────┴─────┐
      ↓           ↓
     是         否
      ↓           ↓
  產生文件    結束對話
```

## 🔄 例外情況

### 可以主動建立文件的情況

1. **重大架構變更**
   - 影響多個模組
   - 需要團隊成員了解

2. **複雜的 Bug 修復**
   - 問題分析複雜
   - 需要記錄根因

3. **重要的設計決策**
   - 需要留下決策記錄
   - 方便未來回顧

**但即使在這些情況下，也應該先詢問使用者！**

## ✨ 最佳實踐

### DO（推薦做法）

✅ 完成功能後簡短說明
✅ 提供測試步驟
✅ 指出注意事項
✅ 詢問是否需要文件
✅ 等待使用者回應

### DON'T（避免做法）

❌ 自動建立總結文件
❌ 自動建立測試文件
❌ 自動建立大量文件
❌ 未經同意就產生文件
❌ 過度使用 Token

## 📝 本規則的由來

**使用者反饋**（2025-11-03）:
> "不要每次做完功能都要做一個總結檔案，也不需要弄一個測試文件，這些檔案文件都需要經過我同意再產出，不要浪費 token"

**解決方案**:
1. 記錄在此文件中
2. AI 助手應遵守此規範
3. 減少不必要的文件產生
4. 節省 Token 額度

---

## 🎓 記憶要點

1. **預設不產生文件** - 除非使用者明確要求
2. **簡短清楚的回應** - 說明修改內容和測試方式
3. **詢問後再行動** - 如果認為需要文件，先詢問
4. **節省 Token** - 避免不必要的文件產生
5. **尊重使用者** - 讓使用者決定是否需要文件

---

**記住這個規則！每次完成功能後，不要自動產生文件，先詢問使用者！**
