## ✅ 已完成功能

### 1️⃣ 後端非同步爬取系統

#### 新增檔案
- **CrawlTaskDTO.java** - 爬取任務 DTO
  - 任務狀態：PENDING、PROCESSING、COMPLETED、FAILED
  - 包含書籍資訊、錯誤訊息、時間戳等

- **ICrawlTaskService.java** - 爬取任務服務介面
  - `createTask()` - 建立任務
  - `getTask()` - 查詢任務狀態
  - `getActiveTasks()` - 取得進行中任務
  - `executeTask()` - 執行任務（非同步）

- **CrawlTaskServiceImpl.java** - 爬取任務服務實作
  - 使用 ConcurrentHashMap 儲存任務狀態
  - @Async 非同步執行爬取
  - 自動清理過期任務（30分鐘）
  - 最多 100 個並行任務限制

- **AsyncConfig.java** - 異步任務配置
  - 核心執行緒數：3
  - 最大執行緒數：5
  - 佇列容量：20
  - 執行緒名稱：crawl-task-*

- **InvCrawlTaskController.java** - 爬取任務 Controller
  - `POST /inventory/crawlTask/create` - 建立任務
  - `GET /inventory/crawlTask/status/{taskId}` - 查詢狀態
  - `GET /inventory/crawlTask/active` - 查詢進行中任務
  - `GET /inventory/crawlTask/batchStatus` - 批次查詢

## 📋 功能流程

### 非同步掃描流程
```
1. 使用者掃描 ISBN
   ↓
2. 前端呼叫 createCrawlTask API
   ↓
3. 後端建立任務（返回 taskId）
   ↓
4. 前端立即提示「已加入佇列」
   ↓
5. 相機保持開啟，可繼續掃描
   ↓
6. 後端非同步執行爬取（@Async）
   ↓
7. 前端每 2 秒輪詢任務狀態
   ↓
8. 爬取完成後彈出通知
   ↓
9. 點擊通知可查看書籍資訊
```

### 拖動功能流程
```
1. 使用者按住浮動按鈕
   ↓
2. 觸發 touchstart 事件
   ↓
3. 記錄初始位置
   ↓
4. touchmove 時更新按鈕位置
   ↓
5. 邊界檢測（不超出螢幕）
   ↓
6. touchend 時儲存位置到 localStorage
   ↓
7. 延遲 100ms 重置拖動狀態（避免觸發點擊）
```

### 新手導航流程
```
1. 首次登入檢查 localStorage
   ↓
2. 未顯示過 → 延遲 1 秒顯示導航
   ↓
3. 畫面灰暗遮罩（75% 不透明度）
   ↓
4. 浮動按鈕高亮（發光動畫）
   ↓
5. 箭頭指向按鈕（彈跳動畫）
   ↓
6. 提示文字「按住可移動位置」
   ↓
7. 點擊任意處關閉導航
   ↓
8. 記錄到 localStorage（不再顯示）
```

## 📊 技術架構

### 後端架構
```
Controller (REST API)
    ↓
Service (業務邏輯)
    ↓
@Async (非同步執行)
    ↓
ConcurrentHashMap (任務儲存)
    ↓
IsbnCrawlerService (爬蟲執行)
```

```

    ├─ 拖動功能
    └─ 新手導航

API Layer (axios)
    └─ crawlTask.js

LocalStorage
    ├─ scan-button-position
    └─ scan-button-guide-shown
```
