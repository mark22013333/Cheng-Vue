## 🧪 測試案例

### 測試 1：一般使用者預約物品

**步驟**：
1. 使用 `testuser` 登入系統
2. 前往「庫存管理」→「物品與庫存管理」
3. 找到可用庫存 > 0 的物品
4. 點擊「預約」按鈕
5. 填寫預約資訊：
   - 預約數量：1
   - 預約開始日期：今天
   - 預約結束日期：30天後
6. 點擊「確認預約」

**預期結果**：
- ✅ 預約成功，提示「預約成功，等待管理員審核」
- ✅ 物品的 `available_qty` 減少，`reserved_qty` 增加
- ✅ 在「借出管理」頁面可以看到預約記錄（狀態：待審核）

---

### 測試 2：管理員審核預約

**步驟**：
1. 使用 `admin` 登入系統
2. 前往「庫存管理」→「借出管理」
3. 找到狀態為「待審核」的預約記錄
4. 點擊「審核」按鈕
5. 選擇「通過」並提交

**預期結果**：
- ✅ 審核通過，狀態變更為「已借出」
- ✅ 物品的 `reserved_qty` 減少，`borrowed_qty` 增加
- ✅ `reserve_status` 從 1（待審核預約）變更為 2（預約通過）

---

### 測試 3：一般使用者歸還自己的借出

**步驟**：
1. 使用 `testuser` 登入系統
2. 前往「庫存管理」→「借出管理」
3. 找到自己的借出記錄（借出人是 testuser）
4. 點擊「歸還」按鈕
5. 選擇歸還狀態（完好/損壞/遺失）
6. 填寫歸還數量和備註
7. 提交

**預期結果**：
- ✅ 可以看到「歸還」按鈕
- ✅ 歸還成功
- ✅ 物品的 `borrowed_qty` 減少，`available_qty` 增加（如果是完好或損壞）

---

### 測試 4：一般使用者不能歸還他人的借出（權限控制）

**步驟**：
1. 使用 `admin` 預約並審核通過一個借出
2. 登出，使用 `testuser` 登入
3. 前往「借出管理」頁面
4. 查看 admin 的借出記錄

**預期結果**：
- ✅ 在 admin 的借出記錄上**看不到**「歸還」按鈕
- ✅ 只有自己的借出記錄才顯示「歸還」按鈕

---

### 測試 5：管理員可以歸還任何人的借出

**步驟**：
1. 使用 `admin` 登入系統
2. 前往「借出管理」頁面
3. 查看所有借出記錄

**預期結果**：
- ✅ 所有借出記錄（無論借出人是誰）都顯示「歸還」按鈕
- ✅ 可以成功歸還任何人的借出

---

### 測試 6：預約時間重疊檢查

**步驟**：
1. 使用 `testuser` 預約物品 A（數量 1，時間：12/4 - 12/10）
2. 等待 admin 審核通過
3. 再次嘗試預約物品 A（數量 1，時間：12/5 - 12/12，時間重疊）

**預期結果**：
- ✅ 第一次預約成功
- ✅ 第二次預約失敗，提示「該物品在選擇的時間段已有預約」

---

### 測試 7：已歸還的物品可以再次預約

**步驟**：
1. 使用 `testuser` 預約物品 A
2. admin 審核通過
3. testuser 歸還物品 A
4. testuser 再次預約物品 A（時間段與之前重疊）

**預期結果**：
- ✅ 歸還後可以再次預約
- ✅ 時間重疊檢查會排除已歸還的記錄

---

### 測試 8：預約記錄不允許修改

**步驟**：
1. 使用 `testuser` 預約物品
2. 在「借出管理」頁面找到這筆預約記錄（status=待審核，reserve_status=1）
3. 查看是否有「修改」按鈕

**預期結果**：
- ✅ 待審核的預約記錄**不顯示**「修改」按鈕
- ✅ 防止修改預約數量導致庫存計算錯誤

## 🔍 後端驗證點

### SQL 檢查

```sql
-- 檢查預約權限是否建立
SELECT menu_id, menu_name, perms, order_num
FROM sys_menu
WHERE perms = 'inventory:management:reserve';

-- 檢查一般使用者角色
SELECT role_id, role_name, role_key, status
FROM sys_role
WHERE role_key = 'common';

-- 檢查測試使用者
SELECT user_id, user_name, nick_name, status
FROM sys_user
WHERE user_name = 'testuser';

-- 檢查測試使用者的角色分配
SELECT u.user_name, r.role_name
FROM sys_user u
JOIN sys_user_role ur ON u.user_id = ur.user_id
JOIN sys_role r ON ur.role_id = r.role_id
WHERE u.user_name = 'testuser';

-- 檢查一般使用者的權限
SELECT m.menu_name, m.perms
FROM sys_role r
JOIN sys_role_menu rm ON r.role_id = rm.role_id
JOIN sys_menu m ON rm.menu_id = m.menu_id
WHERE r.role_key = 'common'
ORDER BY m.order_num;
```
