## 🎯 核心需求

| 需求類別 | 功能描述 |
|---------|---------|
| **形象網站** | 首頁輪播、品牌介紹區塊、精選商品展示 |
| **商品展示** | 分類瀏覽、商品列表、商品詳情頁 |
| **購物流程** | 購物車、結帳、訂單追蹤 |
| **會員系統** | 註冊/登入、LINE/Google 第三方登入、會員中心 |
| **後台管理** | 視覺化管理前台所有內容 |

## ⭐ 設計原則

1. **強制使用 Enum**：所有狀態必須定義 Enum，禁止魔術數字
2. **策略模式**：避免 if-else 堆疊，使用策略模式實現可擴充設計
3. **介面導向**：所有 Service 必須有介面定義
4. **冗餘欄位**：訂單等歷史記錄採用冗餘設計，保留快照
5. **單一職責**：每個類別只負責一件事
